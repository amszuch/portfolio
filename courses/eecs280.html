EECS 280:

Programming and Intro Data Structures

Project 1: Stats

For my first introduction to coding in C++, I was tasked with calculating simple statistical functions such as mean, median, mode, and range of a small set of data. While fairly elementary, this did introduce me to tools such as Xcode as an IDE, Git for version control, and the lldb debugger.

Project 2: Computer Vision

This project was a great opportunity to practice testing, debugging, pointers, strings, streams, I/O and abstract data types in C to create a content-aware image resizer. I was tasked with ensuring member functions of Matrix, Image, and Pixel structs met the given RMEs, calculating an energy and cost matrix, and implementing well-defined seam identification and removal algorithms.


When applied to a ppm image (such as the one above), the final code was able to produce images like the two shown below, maintaining the content by repeatedly removing the minimal vertical seam until the desired dimensions were achieved.

Project 3: Euchre

One of the most infamous projects in EECS. This project combines abstract data types, derived classes, inheritance, polymorphism, and practice with OOP using classes and virtual functions to simulate a classic Midwestern card game. Each of four players is specified as either a human (which takes input) or a simple player (algorithmically-controlled), with the same underlying driver in all scenarios.

Project 4: Web

This lighter-weight project replicated the EECS 280 office hours queue website with an underlying linked list structure and a simple driver to implement the web API. The learning goals included Container ADTs, dynamic memory, The Big Three in C++, constructors, destructors, linked lists, and iterators.

Project 5: Binary Search Tree

The final project in this course involved implementing a map data structure and binary search trees to create a classifier system for Piazza posts. Functors, templates, recursion, binary search trees, and associative containers were all components included in this project. Course staff also specified that certain functions be tree, linear, or tail recursive in their implementation.
